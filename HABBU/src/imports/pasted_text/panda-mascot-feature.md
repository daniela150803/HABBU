Enhance the existing habit-tracking web application by integrating an interactive panda mascot feature into the current UI. Do not redesign the full application and do not change the main navigation or the overall structure. The app already has a visual and functional foundation, so the goal is to add this feature on top of the current experience in a way that feels natural, cohesive, and emotionally engaging.

Main goal

Create a more engaging and memorable user experience by introducing a panda mascot that gives visual feedback, supports the user emotionally, and allows direct interaction. The panda should not be a static decorative element. It must behave like a digital companion that reacts, changes poses and expressions, and becomes an important part of the user journey.

The panda should have a stronger presence across the website, especially in the user profile page, where it should become a central and recognizable element of the experience.

Core experience principles
The panda must feel like an interactive companion, not just a decorative illustration.
The experience should be positive, healthy, and emotionally supportive.
The system should improve retention through pleasant interaction and visual reward, not through pressure.
Do not use streaks, consecutive-day counters, guilt-based messaging, or addictive mechanics.
The feedback should celebrate progress without punishing absence.
1. Interactive completion modal

When the user completes a habit, show a popup modal or overlay featuring the panda as the main character.

This modal should not be just a simple confirmation message. It should be a small interactive moment that feels rewarding and memorable.

Include:
A panda illustration in a celebratory pose.
A short, encouraging message such as:
“Great job!”
“One more step forward”
“Your panda is proud of you”
“Nice progress today”
A primary action button such as:
“Continue”
“Keep going”
“View progress”
A soft entrance animation or microinteraction.
Rounded corners, soft shadows, and a warm, modern visual style.
Important:

The user must be able to interact with the panda, not just read the message.

Examples of interaction:
Clicking the panda triggers a different reaction.
Hovering over the panda activates a subtle animation.
Small interaction buttons can appear, such as:
“Pat the panda”
“Celebrate with panda”
“See another reaction”
“Cheer it up”
Each interaction can trigger:
a different pose,
a facial expression change,
a short animation,
a tiny speech bubble,
or a playful visual response.

The goal is to make the completion of a habit feel emotionally rewarding through interactive feedback, not just visual confirmation.

2. Strong panda presence in the user profile

Strengthen the panda’s role especially in the user profile page.

Add a dedicated panda section or card inside the profile, where the mascot has a clear and prominent presence. This area should help personalize the profile and make it feel more alive.

The panda profile section should include:
A large or highlighted panda illustration.
A short greeting or personalized message from the panda.
A mood or expression that reflects the user’s progress.
A small visual summary of the user’s progress or activity.
Interactive elements that let the user engage with the panda.
The user should be able to interact with the panda in the profile, for example:
Tap or click the panda to trigger different reactions.
Change or cycle through different panda poses.
Reveal different expressions.
Activate a greeting, celebration, or response.
See the panda respond differently depending on user activity.

The panda should feel alive and responsive, not static. It should become a recognizable part of the user’s identity within the app.

The panda must be interactive, not limited to showing static messages.

Design lightweight but meaningful interaction patterns that make the user feel connected to the mascot.

Examples of interaction:

hover to see a subtle movement,
small action chips or buttons like:
“Say hi”
“Pat panda”
“Celebrate”
“See another pose”
“Interact”
brief response bubbles from the panda, such as:
“You’re doing great”
“Nice work”
“Step by step”
“I’m happy for your progress”
“Let’s keep going”

These interactions should be:

simple,
warm,
elegant,
emotionally positive,
and integrated into the UI without overloading it.

The panda should support the experience through soft gamification, but without becoming manipulative or overly childish.

welcome state,

small supportive moments inside the experience.

The panda should appear enough to feel meaningful and memorable, but not so much that it becomes distracting.

7. Important restrictions
Do not redesign the entire application.
Do not replace the current layout or navigation.
Only enhance and extend the current UI.

Create a UI enhancement for the existing habit-tracking web app where the panda becomes an interactive digital companion with:

stronger presence across the experience,
special importance in the user profile,
an interactive completion modal,
multiple poses and expressions,
direct user interaction,
and positive visual feedback.
